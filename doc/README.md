# 嬰兒睡眠與哭聲監測系統 — 專案說明

## 這個專案在做什麼

讓家長在寶寶睡覺時安心做自己的事：系統透過鏡頭與麥克風監測嬰兒床，
當寶寶**哭了**或**可能醒了**，自動發 Telegram 通知到家長手機（附現場截圖）。

## 系統架構

```
【嬰兒房——感測層，零智慧，只搬運資料】
  ESP32-CAM (OV3660) ──── MJPEG 影像串流 (port 81) ──┐
  ESP32 DevKit + INMP441 ─ UDP 音訊串流 (port 5005) ──┤
                                                      ▼
【筆電——運算層，所有智慧集中在這】
  Python 後端 (backend/)
  ├─ 聲音線（主幹）：YAMNet 哭聲偵測 → 分數>門檻持續3秒 → 🚨 哭聲警報
  ├─ 影像線（輔助）：幀差活動量 → 持續2分鐘高活動 → 📱「可能醒了」預警
  └─ 每個警報都附上當下攝影機截圖
                                                      ▼
【通知層】Telegram Bot（兩級通知）
```

## 關鍵設計決策

1. **推論全在筆電，不在 ESP32 上跑模型**
   （曾嘗試 Edge Impulse 燒錄自訓模型，因訓練/部署錄音條件不一致
   （domain shift）而失效，改用筆電上的預訓練模型 YAMNet）
2. **聲音是警報主幹**：哭聲是嬰兒自帶的求救訊號，最可靠；
   影像負責提前預警和提供截圖，判斷權留給看截圖的家長。
3. **麥克風用 INMP441（I2S 數位）而非 MAX9814（類比）**：
   繞過 ESP32 低品質 ADC、無 AGC 干擾、取樣率硬體保證精確。
4. **時間濾波＋冷卻**：滑動視窗多數決才觸發、警報後冷卻 120 秒防轟炸。
5. **先支援有光源環境**，夜視硬體（無 IR 濾光片魚眼鏡頭＋850nm 補光燈）
   到貨後擴展為 24 小時版，架構不變、只補一組夜間校準參數。

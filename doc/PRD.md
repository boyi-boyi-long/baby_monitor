# PRD — 嬰兒睡眠與哭聲監測系統

## 1. 背景與問題

家長在寶寶睡覺時需要處理家事、工作或休息，無法一直守在嬰兒床旁。
現有痛點：

- 不知道寶寶什麼時候醒、什麼時候哭，只能反覆跑去查看；
- 市售智慧監視器（如 Nanit）價格昂貴，且影像資料上傳雲端有隱私疑慮；
- 曾嘗試自製方案（ESP32 + Edge Impulse 自訓哭聲模型）但部署後偵測失效。

## 2. 產品目標

用手邊既有硬體（ESP32-CAM、ESP32 DevKit、筆電）打造一套
**全區域網路運作、資料不出家門**的嬰兒監測系統：

> 寶寶哭了或可能醒了 → 家長的 Telegram 立刻收到通知＋現場照片。

## 3. 目標用戶

- 主要：本專案開發者自己的家庭（自用產品）；
- 次要：未來可販售給有相同需求的 maker 家長。

## 4. 核心使用情境（User Stories）

| # | 情境 | 驗收標準 |
|---|---|---|
| US-1 | 寶寶在床上哭了，我在客廳做家事 | 哭聲持續約 3 秒內，手機收到 🚨 警報＋現場截圖 |
| US-2 | 寶寶醒了在床上翻動但還沒哭 | 持續活動約 2 分鐘後，手機收到 📱 預警「可能醒了」＋截圖 |
| US-3 | 寶寶只是睡眠週期間短暫扭動、翻身 | **不通知**（2 分鐘內歸於平靜即不觸發） |
| US-4 | 系統本身故障（鏡頭或麥克風斷線） | 斷線超過 1 分鐘，手機收到 ⚠️ 系統離線通知 |
| US-5 | 寶寶連續哭，我已經在處理了 | 警報觸發後 120 秒冷卻，不重複轟炸 |
| US-6 | 我人在外面，臨時想看一眼寶寶現在的樣子 | Telegram 傳 `/photo`，幾秒內收到現場截圖 |
| US-7 | 我人在外面，想知道寶寶現在是在睡還是可能醒了，不只是看照片自己判斷 | Telegram 傳 `/status`，收到文字判斷（安靜／可能醒了）＋現場截圖 |
| US-8 | 第一次設定好 Bot，不知道有哪些指令可以用 | 開機自動收到說明訊息；之後隨時傳 `/help` 可再查一次 |

## 5. 功能需求（按開發階段）

### 階段 1：哭聲警報機（MVP）✅ 程式完成
- F1-1 ESP32+INMP441 以 16kHz 串流音訊到筆電（UDP）
- F1-2 筆電以 YAMNet 預訓練模型偵測哭聲，每 0.5 秒判斷一次
- F1-3 滑動視窗多數決（6 次中 4 次超門檻）觸發警報
- F1-4 Telegram 發送文字警報
- F1-5 音訊節點離線偵測與通知

### 階段 2：警報附截圖 ✅ 程式完成
- F2-1 警報觸發時自動從 ESP32-CAM 抓取當下畫面附在通知中
- F2-2 截圖失敗不影響文字警報發送
- F2-3 定時拍照回報：每 15 分鐘自動傳一張現場畫面到 Telegram（不管有沒有警報）
- F2-4 手動拍照指令：在 Telegram 傳 `/photo` 給 Bot，立即回傳一張現場截圖
- F2-5 狀態查詢指令：在 Telegram 傳 `/status` 給 Bot，回報目前是安靜／可能醒了＋現場截圖
- F2-6 使用說明：程式開機自動發一次上線通知＋指令列表；`/help` 或 Telegram 內建 `/start` 按鈕隨時可查

### 階段 3：活動量預警 ✅ 程式完成
- F3-1 持續讀取 MJPEG 串流計算幀差活動量
- F3-2 活動量持續超標 2 分鐘 → 低級預警通知＋截圖
- F3-3 短暫活動自動歸零，不誤報（對應 US-3）

### 階段 4：校準模式
- F4-1 LOG_ONLY 模式：只記錄分數/活動量到 CSV，不發通知
- F4-2 以 3~5 天真實資料決定正式門檻

### 階段 5：夜間擴展（等硬體到貨）
- F5-1 更換無 IR 濾光片魚眼鏡頭＋850nm 紅外補光
- F5-2 夜間環境重新校準一組門檻參數

## 6. 非功能需求

| 類別 | 需求 |
|---|---|
| 隱私 | 影像與音訊**不離開區域網路**；僅警報截圖經 Telegram 發送 |
| 可靠性 | 後端須 24 小時常駐；串流斷線自動重連；程式崩潰可自動重啟 |
| 資源 | 筆電 CPU 即可運行（無需 GPU）；記憶體用量長期穩定不增長 |
| 誤報率 | 校準後目標：每晚誤報 ≤ 1 次（由階段 4 資料驗證） |
| 延遲 | 哭聲發生到手機收到通知 ≤ 10 秒 |

## 7. 明確不做（Out of Scope）

- ❌ 不在 ESP32 上跑任何 ML 模型（已被 domain shift 教訓過）
- ❌ 不做雲端儲存、遠端（非區網）觀看
- ❌ 第一版不做 MediaPipe 姿勢分類（由「預警＋截圖給人判斷」覆蓋）
- ❌ 不做呼吸監測等醫療級功能（本產品非醫療器材，不能取代大人照護）

## 8. 成功指標

1. 連續運行 7 天不崩潰、記憶體不增長；
2. 實際哭聲事件 100% 在 10 秒內通知（漏報 = 最嚴重失敗）；
3. 校準後誤報每晚 ≤ 1 次；
4. 家長主觀評價：「敢放心離開房間做事」。

## 9. 風險與對策

| 風險 | 對策 |
|---|---|
| YAMNet 對自家環境誤判率高 | 階段 4 校準；必要時升級為 YAMNet embedding＋自訓分類頭（遷移學習） |
| Wi-Fi 不穩導致斷流 | 兩端皆有自動重連；離線超時通知（F1-5） |
| 夜間硬體效果不如預期 | 備案：小夜燈方案；850nm 補光實測後再定案 |
| 家人誤觸/寵物入鏡誤報 | 校準期收集真實負樣本情境調整門檻 |
